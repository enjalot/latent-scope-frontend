        ## Latent Scope

        Once my dataset is in this format, I can upload it to Latent Scope, which takes me to the Setup Process. To run the Setup Process for your own datasets, you need to install Latent Scope [(instructions here)](https://enjalot.github.io/latent-scope/install-and-config) via `pip install latentscope` or by building it from source [Github link](https://github.com/enjalot/latent-scope).

         The Setup Process takes the dataset and converts it into a format that is suitable for "visual exploration". Here's a GIF of what that looks like for the Founders Dataset:

        

        I'm going to cover the details of the Setup Process for those who are already familiar with Machine Learning terminology. If you're unfamiliar with the terminology, feel free to skip this section, and pick up from the Explore Interface, which is the main focus of this post.

        ### The Setup Process

        The are 5 steps to the Setup Process:

        1.  Calculate <span style={{ color: '#FF9B50', fontWeight: 'bold' }}>embeddings</span> for a chosen text column of each row in our dataset (in our case, we chose the <span style={{ color: '#4CAF50', fontWeight: 'bold' }}>highlight</span> column)
        2.  <span style={{ color: '#FF9B50', fontWeight: 'bold' }}>Project</span> those embeddings down to two-dimensional coordinates using <a href="https://umap-learn.readthedocs.io/en/latest/index.html" target="_blank">UMAP</a>
        3.  <span style={{ color: '#FF9B50', fontWeight: 'bold' }}>Cluster</span> those coordinates via the <a href="https://hdbscan.readthedocs.io/en/latest/api.html" target="_blank">HDBSCAN</a> clustering algorithm
        4.  Generate a representative <span style={{ color: '#FF9B50', fontWeight: 'bold' }}>label</span> for all the highlights in each cluster using an LLM
        5.  Create a <span style={{ color: '#FF9B50', fontWeight: 'bold' }}>scope</span> that packages everything together for the Explore Interface

        <div style={{padding: '10px 0px'}}>
            <img src="/latent-scope/setup.png" className={styles.diagram} alt="Setup Process Options" />
        </div>        

        The Setup Process lets you choose amongst different options for each of these steps. For example, you can specify which embedding model to use, customize the UMAP and HDBSCAN parameters beyond the provided defaults, and choose which LLM to use to generate the labels.

        For the examples you'll see in this post, I chose [jina-embedding-v3](https://huggingface.co/jinaai/jina-embeddings-v3) as the embedding model and [gpt-4o mini](https://platform.openai.com/docs/models/gpt-4o-mini) from OpenAl as the LLM to generate the labels. (gpt-4o mini requires an OpenAI API key - it cost me less than a cent to generate these labels)

        The Setup Process is designed to support experimentation.         Here's a screenshot of what the Founders Dataset looks like when viewed in the Explore Interface. Hover over any row in the table to see the point's location in the visual map.

        We see 3 primary UI components:

        1.  The <span style={{color: '#FF9B50', fontWeight: 'bold'}}>table</span> on the left. This contains all the columns in our original Founders dataset, plus additional columns that were added during the Setup process.
        2.  Above the table, we see the <span style={{color: '#FF9B50', fontWeight: 'bold'}}>Search Bar</span>
        3.  The <span style={{color: '#FF9B50', fontWeight: 'bold'}}>visual map</span> of the dataset on the right.

        For the purposes of this post, here's what you need to know about this "visual map":

        * The map consists of a bunch of points, one for each individual highlight in our Founders dataset.
        * These points are arranged such that highlights with "similar meanings" sit next to each other in the map (i.e. these points are UMAP-ed embeddings)
        * These points are then automatically grouped into clusters based on their positions in the map
        * Each cluster is assigned a label. This label is generated by an LLM which reads each highlight in a cluster and generates it as a representative label.

        By default, the table displays the first 20 rows in our dataset, and their corresponding locations in the visual map are bolded in green. Hover over any row in the table to see the point's location in the visual map.

        ## General Takeaways

        I showed above how I use Latent Scope as a way to visually guide me towards the books that are most interesting to me, but I think this use case generalizes into a broader use case, where:

        * You have a collection of primary sources (books, research papers)
        * You extract a bunch of "representative samples" (abstracts, highlights, quotes) from those primary sources. You can have LLMs do this for you.
        * Each of these representative samples becomes a row in your dataset, along with a link to the original primary source + any relevant metadata (see schema below)
        * You create a scope from this dataset, embedded on the "representative sample" column.

        Once you have this dataset, you can upload it to Latent Scope, and use Latent Scope to visually explore your primary sources, which I can imagine being an excellent tool for doing research, or even just following your own curiosity.

        What's nice about this approach is that you can extract as many representative samples as you want. In the Founders dataset, I don't mind that there are a lot of highlights that are irrelevant to my interests - such as the highlights in the cluster that aren't exactly related to a love for reading. The ones that I do manage to find that are relevant matter a lot more!
